Signal Lab · the workshop notes

How a feeling becomes a signal

Every piece in this lab is synthesized from first principles — no commercial samples, no presets. This is the full record of how each one is made: the source material, the exact DSP ingredients, the transformations applied, and the SuperCollider internals that voice them in realtime.

2 engines · numpy + SuperCollider 44.1 kHz · float64 4 creations AI · text→patch
01

The process, end to end

A creation is never written as audio directly. It moves through a chain — feeling first, then a small reusable description of a sound (a patch), then the engine that renders it, then a composition that arranges many sounds in time, and finally either an offline master or a realtime performance and a visualizer. The same patch format feeds both engines and the AI.

A patch can be hand-written, designed in a browser bench, or generated by the AI from a sentence. Whichever way it arrives, it is the shared language: Python renders it offline, and SuperCollider voices it live.

The "organs" pattern

Every composition is built as a dictionary of organs — named stereo voices, each with a level — that are summed, soft-clipped, and written out. Movement comes from automation lanes (S.curve([(t, value), …])): per-sample curves that push a cutoff open, swell an amplitude, or cross-fade one color into another across the piece.

# the shape every piece shares
organs = {
    'pad':  (pad(lanes),  0.70),
    'bass': (bass(lanes), 0.72),
    'lead': (lead(lanes), 0.60),
}
mix = sum(level * as_stereo(sig) for sig, level in organs.values())
mix = soft_clip(mix * 0.6)            # analog-style ceiling
write_wav('piece.wav', mix)
02

The ingredients — synth.py

The engine is plain numpy: every signal is a float64 array in roughly [−1, 1]. Oscillators are band-limited by additive synthesis — built as a sum of sine harmonics up to Nyquist, so there is zero aliasing and the harmonic content is exactly what you wrote. The filter is an RBJ-cookbook biquad run block-wise so a swept cutoff stays click-free.

Oscillators

IngredientWhat it is
sine / saw / squareBand-limited primitives. Saw = harmonics 1/k; square = odd harmonics 1/k.Phase is integrated from instantaneous frequency, so pitch can glide or vibrato within one note.
supersaw(voices, detune)Several detuned saws summed.Slow beating between voices = the lush, moving string pad. The Jarre body.
additive(freq, amps)Explicit harmonic recipe — you give the amplitude of each partial.Used for hollow flutes, bells, organ pedals — timbre by drawing the spectrum.
fm2(ratio, index)Two-operator FM (phase modulation).Integer ratios → glassy/plastic; inharmonic ratios → bells and metal. The "digital" timbre.
noise(seed)White noise.Raw material for wind, breath, claps, risers — pitched by a bandpass.

Envelopes & modulation

IngredientWhat it is
adsr(a,d,s,r)Linear attack/decay/sustain/release shaping per note.
lfo(rate, shape)Low-frequency oscillator (sine/tri/saw) for cyclic motion.
drift(amount, rate)Smoothed random multiplier around 1.0.The "analog is alive" factor: tiny continuous pitch/param wander. Its absence is what makes a sound read as digital.
curve([(t,v),…])Piecewise-linear automation lane — how a parameter moves across a whole piece.

Resonant filter (RBJ biquad, block-wise)

Four responses — resonant_lpf / hpf / bpf / notch — each accepting a scalar or a per-sample cutoff. Coefficients update every 256 samples and filter state carries across blocks, so a cutoff that sweeps from 300 Hz to 12 kHz has no zipper noise.

# swept cutoff, click-free: recompute coeffs per block, carry state
for start in range(0, n, block):
    b, a = _biquad_lpf(mean(cutoff[start:end]), q)
    out[start:end], zi = lfilter(b, a, x[start:end], zi=zi)

Space & effects

IngredientRole
phaserCascade of swept all-pass stages → moving notches. The Oxygène string shimmer (Jarre ran his Eminent organ through a Small Stone phaser).
reverb / reverb_stFFT convolution with a decaying-noise impulse. Stereo uses two decorrelated IRs (L/R) → an enveloping room, not a mono wash.
pingpong / tape_delayRhythmic echoes (alternating L/R), and a feedback delay with tape-flutter wobble.
shimmer_haloReverb → pitch the wet up an octave → reverb again: a rising angelic halo.
soft_clip(drive)tanh saturation — gentle analog warmth, and the master ceiling on every mix.
pan / stereo_width / autopanConstant-power placement, mid/side width (0 = mono, >1 = wider), and slow L↔R motion.

Sampling — turning real recordings into instruments

IngredientRole
load_audioDecode any mp3/wav to a float array at 44.1 kHz (via ffmpeg), peak-normalized.
resample_speedPlay slower/lower or faster/higher (the "abyss" pitch-down on the Jupiter sample).
granular_cloudChop a recording into windowed grains, scatter them in stereo → an evolving texture/drone.
spectral_droneTake a segment's magnitude spectrum, re-excite it with noise → a sustained tone carrying the sample's color, at any length.
convolve_withUse a recording as an impulse response — imprint its spectral/temporal character onto another sound.
bitcrush(bits, rate)Quantize amplitude (bit depth) and decimate sample rate.The literal sound of digitization — the core of Vitrine.
ringmodMultiply by a sine carrier → inharmonic, metallic, alien.

Pitch & scale

note_to_hz('A4') → 440. A scale(root, mode) returns the comb of allowed pitches so a melody can never land on a wrong note. Modes include major, minor, dorian, phrygian, lydian and wholetone — the last has no semitones, so it floats and cannot cadence (used deliberately as the "no home" color).

03

The patch bridge & the AI

A patch is a small JSON ingredient — one oscillator, one filter, an envelope, an LFO, and effects, with an optional step sequence. patch.py renders it to stereo through the engine; the same file is read by SuperCollider and written by the AI.

{ "osc":    {"type":"wave", "wave":"saw", "voices":2, "detune":12},
  "filter": {"type":"lowpass", "cutoff":1500, "q":0.8},
  "env":    {"a":0.4, "d":0.2, "s":0.85, "r":0.5},
  "lfo":    {"target":"filter", "rate":0.1, "depth":0.3},
  "fx":     {"reverb":0.5, "width":0.7} }

Oscillator types: wave (saw/square/sine/triangle, optional unison), additive, fm, ping (a struck pitch-drop sounding), and stardust (sparse decaying sine twinkles). Filter types: lowpass / highpass / bandpass / notch.

Never trust raw model JSON

The AI uses OpenRouter (default a cheap model, openai/gpt-4o-mini) to turn a sentence into a patch. Every field is then passed through validate_patch(), which clamps and coerces every value into a safe, renderable range — so a hallucinated cutoff of 90 000 or a missing field can never produce a broken or dangerous sound. There is also analyze(sig), which measures a rendered sound (rms, peak, spectral centroid, brightness, texture) and can turn one of our own creations back into a prompt seed.

04

The realtime engine — SuperCollider

For live playing, the same patch format drives scsynth. engine.scd defines a set of SynthDefs that mirror the patch oscillator types; sc_bridge.py stays the brain, translating a patch into an OSC /s_new message and scheduling sequences. Audio comes out of SuperCollider, not over HTTP. OSC port 57110.

The shared tail

Every voice ends in one shared signal path: select filter response → filter envelope & LFO → amplitude envelope → a per-voice tanh (so no single voice can blow up) → FreeVerb → stereo pan. A master limiter sits at the very tail of the node tree so summed voices can never clip into noise.

// the safety chain shared by all voices
sig = Select.ar(ftype, [RLPF, RHPF, BPF].(sig, co, rq));
sig = sig * env * amp;
sig = sig.tanh;                       // per-voice ceiling
sig = FreeVerb.ar(sig, mix, room: 0.7);
Out.ar(0, Pan2.ar(sig, pan));
// …and at the tail of the tree:
Limiter.ar(LeakDC.ar(In.ar(0,2)) * 0.7, 0.9, 0.05);

The voices

How a patch is routed

osc.type→ SynthDefSafety applied in the bridge
ping\labping—
stardust\stardustdensity / lo / hi passed through

Resonance is always sent as reciprocal-Q clamped to 1/clip(q, 0.8, 8) so RLPF/RHPF can never self-oscillate into a scream. A native GUI panel, control.scd (EZSliders + FreeScope), lets you tweak each instrument live.

05

The creations

Four pieces, each a different question. For every one: where the sound came from, how it was transformed, and — where it applies — how it is voiced through SuperCollider.

The Black Theaterdream
key A Phrygian · tempo 60 BPM · ~96 s · dream.py + theater_sc.py

Source material: a childhood dream — a child alone on a stage, watched by an audience of unseen eyes. There is no recording; the source is pure synthesis. The three movements follow the eyes: I — kind, II — adoration & envy, III — the eyes close.

The organs

  • pedal — the stage floor, an organ pedal tone
  • audience — the eyes; a wide, watching drone
  • child — a breathy hollow flute, lonely and forward
  • fishing — soundings into the void (a struck ping)
  • footsteps — uneven human steps onto the stage
  • stardust — analog cosmic glitter, woven throughout
  • adore / envy — the swelling and the souring in movement II

The SuperCollider re-voicing

theater_sc.py runs the full pipeline: each organ's character is written as a sentence → the AI designs a patch → validate_patch → played live through scsynth. Drones, the child, and the glint are then overridden with safe envelopes and dark cutoff caps so the AI's color is kept but never its glare.

What we fixed: early SC mixes turned to white noise mid-piece — solved with the master limiter, the reciprocal-Q clamp, and an FM index cap. A sustained high bell became a "mosquito" and was replaced with the softer stardust; a naked held sine became the struck fishing ping; footsteps were made intentional and beautified. Editions in versions/theater-sc-v001/.

Icarus, or the Effort of Flightflight
key A → A Lydian · tempo 128 BPM · 180 s · icarus.py

Source material: a flying dream — clapping the hands fast to stay just above the rooftops, the feeling great but the effort hard to hold — fused with cosmic horror and built on a real recording of Jupiter's ion-acoustic plasma waves (space_library/jupiter_ion_acoustic.mp3).

How the Jupiter sample is engineered

  • resample_speed(0.35) → a slow, low abyss
  • granular_cloud → a shifting void cloud
  • spectral_drone → a sustained drone in the sample's color
  • reverse + filter sweep → the riser into the breakthrough

The dramatic arc

  • Struggle (0–45 s) — dissonant tension, fighting to rise
  • Breakthrough (~46 s) — a held breath, then it bursts into bright Lydian
  • Keep the pace — driving bass + locked clap; altitude soaring yet wavering
The key device: an effort automation lane that alternates flap (clap on, bass driving) and glide (clap off — you float, and wind rushes past). The lead floats on altitude, not effort, so it keeps singing even when the hands rest. A master dip-and-surge gates the breakthrough breath.

Homage to Oxygènestudy
key A minor · tempo 104 BPM · 40 bars · oxygene.py

Source material: the Jarre signature itself, rebuilt from scratch in the real engine — the spirit, not a sample. A study in where our sound comes from.

The voices

  • pads — detuned saws → phaser (the Eminent-310 + Small Stone)
  • arp — a bubbling 16th-note arpeggio (the hypnotic motion)
  • bass — a pulsing octave bass
  • lead — a singing portamento melody (the soul)
  • perc — a soft electronic kick / snare / hat

The portamento lead

The melody is rendered as one continuous frequency array that glides exponentially between notes over ~50 ms, with gentle vibrato — so the lead sings rather than steps. This is the engine's offline answer to SuperCollider's \labglide.

Vitrineconverter loop
key D minor vs C whole-tone · ~100 BPM grid · 300 s · vitrine.py

Source material: a photograph of shop-window mannequins — stylized soulless souls; the digitized image of a person with the person gone. The structure is the signal chain: a warm phrase put through an ADC and never fully recovered.

The converter, pass by pass

One analog phrase is rendered once, then re-converted worse each pass: anti-alias filter → sample/hold → quantize (16→10→7→5→3→2 bit) → reconstruction filter. Quantization error is irreversible — that is the soul we lose. The lead literally migrates from D-minor toward whole-tone as it digitizes.

The substrate that doesn't degrade

Under everything, a stardust bed at free pitches that is never sent through the converter — so it doesn't quantize. When the soul is finally gridded to nothing, the dust is still shimmering. The form is lost; the matter is conserved.

Lineage: Basinski's Disintegration Loops (the same loop re-digitized, losing material every pass) · Alva Noto / Ryoji Ikeda (the aesthetics of error — sine pips, grid clicks, quantization hiss) · Jarre's Revox trick (dry one side, delayed the other) is the glass reflection. The piece ends unresolved — the DAC reaches back at 3:34, almost becomes warm again, fails, and we keep going. Full story: VITRINE.md.

07

Running it yourself

Render a piece offline, or serve the whole lab and play live.

# render a creation to WAV
./.venv/bin/python vitrine.py        # or dream.py / icarus.py / oxygene.py

# serve the lab (portal, benches, visualizer) at 127.0.0.1:7700
./.venv/bin/uvicorn server:app --port 7700

# realtime: open engine.scd in SuperCollider, evaluate it, then
./.venv/bin/python theater_sc.py     # performs The Black Theater live
